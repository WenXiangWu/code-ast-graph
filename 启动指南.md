# Code AST Graph 启动指南

## 快速启动

### 方式一：一键启动脚本（推荐）
```bash
# Windows
.\start.bat

# Linux/Mac
./start.sh
```

### 方式二：手动启动
```bash
# 1. 启动 Neo4j
docker-compose up -d

# 2. 等待 Neo4j 完全启动（约 30-40 秒）
docker-compose logs -f neo4j

# 3. 启动后端服务
python run.py

# 4. 启动前端（可选）
cd frontend
npm run dev
```

## 数据持久化

### Docker 卷说明

| 卷名 | 挂载点 | 用途 | 大小 |
|------|--------|------|------|
| `neo4j-data` | `/data` | 数据库文件（节点、关系、索引） | ~几 GB |
| `neo4j-logs` | `/logs` | 日志文件 | ~几 MB |
| `neo4j-plugins` | `/plugins` | APOC 等插件 | ~20 MB |

### 为什么数据会丢失？

**之前的问题**：
```bash
# ❌ 错误方式：没有挂载数据卷
docker run -d --name jqassistant-neo4j \
  -p 17687:7687 \
  neo4j:4.4

# 删除容器时，数据也被删除
docker rm jqassistant-neo4j  # 💥 数据丢失！
```

**现在的方式**：
```yaml
# ✅ 正确方式：使用 Docker Compose + 数据卷
services:
  neo4j:
    volumes:
      - neo4j-data:/data        # 数据持久化
      - neo4j-plugins:/plugins  # 插件持久化
```

### 数据卷管理

```bash
# 查看所有卷
docker volume ls

# 查看卷详情
docker volume inspect neo4j-data

# 备份数据卷
docker run --rm -v neo4j-data:/data -v d:/backup:/backup ubuntu tar czf /backup/neo4j-backup.tar.gz /data

# 恢复数据卷
docker run --rm -v neo4j-data:/data -v d:/backup:/backup ubuntu tar xzf /backup/neo4j-backup.tar.gz -C /

# 完全清理（包括数据）
docker-compose down -v
```

## APOC 插件

### 为什么每次都下载？

**之前的问题**：
- 没有挂载 `/plugins` 目录
- 每次容器重启，APOC 插件被删除
- Neo4j 启动时检测到插件不存在，重新下载（约 20MB，需要 10-30 秒）

**现在的解决方案**：
```yaml
volumes:
  - neo4j-plugins:/plugins  # ✅ 插件持久化
```

**行为**：
- ✅ **首次启动**：下载 APOC 插件到 `/plugins/apoc.jar`（约 30 秒）
- ✅ **后续启动**：直接使用已下载的插件（约 5 秒）

## 索引持久化

### 索引存储位置
索引存储在 `/data/databases/neo4j/schema/index/` 目录中，属于数据库文件的一部分。

### 为什么每次启动都创建索引？

**实际情况**：
```python
# backend/main.py 中的逻辑
CREATE INDEX idx_method_signature IF NOT EXISTS
FOR (m:Method) ON (m.signature)
```

**行为说明**：
- ✅ **数据卷存在 + 索引已创建**：
  ```
  2026-02-26 10:16:03,949 - INFO - ✅ MCP 查询索引已存在，无需创建
  ```
  
- ✅ **数据卷丢失 / 首次启动**：
  ```
  2026-02-26 10:16:03,949 - INFO - ✅ 创建索引: idx_method_signature (Method.signature)
  2026-02-26 10:16:04,024 - INFO - ✅ 创建索引: idx_method_name (Method.name)
  ...
  2026-02-26 10:16:42,948 - INFO - ✅ 成功创建 12 个 MCP 查询索引
  ```

**结论**：
- 索引**已经持久化**在 `/data` 目录中
- 后端代码使用 `IF NOT EXISTS`，所以是**幂等操作**
- 如果看到"创建索引"日志，说明数据卷是新的或被清空了
