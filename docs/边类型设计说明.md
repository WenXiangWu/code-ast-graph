# 知识图谱边类型设计说明

## 边类型总览

### 1. 方法调用相关（Method → Method）

| 边类型 | 说明 | call_type 属性 | 示例 |
|--------|------|----------------|------|
| **CALLS** | 普通方法调用 | `Internal`, `Resource`, `Autowired` | `ServiceA.methodA --CALLS[Resource]--> ServiceB.methodB` |
| **DB_CALL** | 数据库操作调用（调用 Mapper 方法） | `Internal`, `Resource`, `Autowired` | `ServiceImpl.save --DB_CALL[Resource]--> UserMapper.insert` |
| **DUBBO_CALLS** | Dubbo 方法调用（跨项目 RPC） | `Reference`, `DubboReference` | `ServiceA.process --DUBBO_CALLS[Reference]--> RemoteService.query` |

**优先级**：
1. 如果被调用类是 `MAPPER` → 使用 `DB_CALL`
2. 否则，如果 `call_type` 是 `Reference`/`DubboReference` → 使用 `DUBBO_CALLS`
3. 否则 → 使用 `CALLS`

### 2. MQ 消息队列相关

| 边类型 | 起点 | 终点 | 说明 | 示例 |
|--------|------|------|------|------|
| **MQ_KAFKA_CONSUMER** | Class | MQ_TOPIC | 类消费 Kafka topic | `LotteryMqConsumer --MQ_KAFKA_CONSUMER--> TOPIC_LOTTERY_STAR_BOX` |
| **MQ_ROCKET_CONSUMER** | Class | MQ_TOPIC | 类消费 RocketMQ topic | `MusicOfflineConsumer --MQ_ROCKET_CONSUMER--> CHATROOM_MUSIC_OFFLINE_PLAY` |
| **MQ_CONSUMES_METHOD** | MQ_TOPIC | Method | Topic 由哪个方法消费 | `TOPIC_LOTTERY_STAR_BOX --MQ_CONSUMES_METHOD--> consumerStarBoxLottery` |
| **MQ_KAFKA_PRODUCER** | Method | MQ_TOPIC | 方法发送到 Kafka | `updateRoom --MQ_KAFKA_PRODUCER--> CHATROOM_ENTER_READY` |
| **MQ_ROCKET_PRODUCER** | Method | MQ_TOPIC | 方法发送到 RocketMQ | `sendToQueue --MQ_ROCKET_PRODUCER--> CHATROOM_MUSIC_OFFLINE_PLAY` |

**消费链路**：
```
Class --MQ_KAFKA_CONSUMER--> MQ_TOPIC --MQ_CONSUMES_METHOD--> Method
```

**生产链路**：
```
Method --MQ_KAFKA_PRODUCER--> MQ_TOPIC
```

### 3. Dubbo 服务相关

| 边类型 | 起点 | 终点 | 说明 | 示例 |
|--------|------|------|------|------|
| **DUBBO_DEPENDS** | Class | Interface | 类依赖 Dubbo 服务（通过 @Reference/@DubboReference 注入） | `ChatroomUpdateController --DUBBO_DEPENDS[roomCmdRemoteService]--> RoomCmdRemoteService` |
| **DUBBO_PROVIDES** | Class | Interface | 类提供 Dubbo 服务实现 | `RoomCmdServiceImpl --DUBBO_PROVIDES--> RoomCmdRemoteService` |
| **DUBBO_CALLS** | Method | Method | 方法调用外部 Dubbo 服务方法 | `process --DUBBO_CALLS[via=remoteService]--> RemoteService.query(...)` |

**注意**：
- **DUBBO_DEPENDS**：类级别的依赖关系（14 个）
- **DUBBO_CALLS**：方法级别的调用关系（330 个）

### 4. 数据库相关

| 边类型 | 起点 | 终点 | 说明 |
|--------|------|------|------|
| **MAPPER_FOR_TABLE** | MAPPER | Table | Mapper 操作的数据库表 |
| **DB_CALL** | Method | Method | 方法调用 Mapper 方法（数据库操作） |

### 5. 其他结构关系

| 边类型 | 起点 | 终点 | 说明 |
|--------|------|------|------|
| **CONTAINS** | Project | Class/Interface/MAPPER/ARIES_JOB | 项目包含类 |
| **DECLARES** | Class | Method/Field | 类声明方法或字段 |
| **IMPLEMENTS** | Class | Interface | 类实现接口 |
| **EXTENDS** | Class | Class | 类继承父类 |
| **EXPOSES** | Method | RpcEndpoint | 方法暴露 RPC/HTTP 入口 |
| **EXECUTES_JOB** | Method | Job | 方法执行定时/延时任务 |

## 边类型决策树

```
Method 调用 Method:
  ├─ 被调用类是 MAPPER？
  │   └─ 是 → DB_CALL
  └─ 否 → call_type 是什么？
      ├─ Reference/DubboReference → DUBBO_CALLS
      ├─ Resource/Autowired → CALLS
      └─ Internal → CALLS
```

## 重要说明

1. **同一对节点只保留一条边**：
   - 如果是 Mapper 调用，只创建 `DB_CALL`，不创建 `CALLS`
   - 如果是 Dubbo 调用，只创建 `DUBBO_CALLS`，不创建 `CALLS`

2. **MQ 发送不创建 CALLS**：
   - `kafkaProducer.send(topic, msg)` 不创建到 `send` 方法的 `CALLS`
   - 而是创建 `Method --MQ_KAFKA_PRODUCER--> MQ_TOPIC`

3. **跨项目判断**：
   - `Reference`/`DubboReference` 且被调用方在不同项目 → `DUBBO_CALLS`
   - `Reference`/`DubboReference` 但被调用方在同项目 → `Internal`（降级为 `CALLS`）

4. **外部服务**：
   - 如果被调用的类不在任何 Project 中（`is_external = true`），仍然创建 `DUBBO_CALLS`
