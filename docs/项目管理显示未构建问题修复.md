# 项目管理显示"未构建"问题修复说明

## 问题描述

用户反馈:
- 扫描日志显示 `official-core-pro-web` 构建成功
- Neo4j 中也存在数据
- 但项目管理页面显示"未构建"

## 问题分析

### 1. 根本原因

**Scanner V2 和后端 API 使用的节点类型不一致**:

- **Scanner V2** (新版): 创建 `Repo` 节点
- **后端 API**: 查询 `Project` 节点

```python
# Scanner V2 创建 Repo 节点
def _create_repo_node(self, project_name: str, project_path: str):
    self.client.execute_write("""
        MERGE (r:Repo {name: $name})
        SET r.path = $path, r.scanned_at = datetime()
    """, {'name': project_name, 'path': project_path})

# 后端 API 查询 Project 节点 (旧版)
@app.get("/api/projects")
async def get_projects():
    result = neo4j_storage.execute_query("""
        MATCH (p:Project)  # ❌ 查询不到 Repo 节点
        RETURN p.name as name, ...
    """)
```

### 2. 验证结果

运行 `check_repo_nodes.py` 确认:

```
检查 Repo 节点:
  ✅ 找到 1 个 Repo 节点: official-core-pro-web
    - Types: 407
    - Methods: 0 (需要通过 Type-[:DECLARES]->Method 查询)
    - Fields: 0 (需要通过 Type-[:DECLARES]->Field 查询)
    - Packages: 95

检查 Project 节点:
  ❌ 没有找到 Project 节点
```

### 3. 图结构差异

**Scanner V2 的图结构**:
```
Repo -[:CONTAINS]-> Package
Repo -[:CONTAINS]-> Type
Type -[:DECLARES]-> Method
Type -[:DECLARES]-> Field
```

**Scanner V1 的图结构** (旧版):
```
Project -[:CONTAINS]-> Type
Project -[:CONTAINS]-> Method
Project -[:CONTAINS]-> Field
```

## 修复方案

### 修复 1: 更新 `/api/projects` 接口

支持同时查询 `Repo` 和 `Project` 节点,优先查询 `Repo`:

```python
@app.get("/api/projects")
async def get_projects():
    """获取 Neo4j 中已构建的项目列表（兼容旧接口，支持 Repo 和 Project 节点）"""
    # 先查询 Repo 节点 (Scanner V2)
    result = neo4j_storage.execute_query("""
        MATCH (r:Repo)
        RETURN r.name as name, COALESCE(r.path, '') as path,
               COALESCE(r.scanned_commit_id, '') as scanned_commit_id,
               r.scanned_at as scanned_at
        ORDER BY r.name ASC
    """)
    
    # 如果没有 Repo 节点,再查询 Project 节点 (Scanner V1)
    if not result:
        result = neo4j_storage.execute_query("""
            MATCH (p:Project)
            RETURN p.name as name, ...
        """)
```

### 修复 2: 更新 `/api/projects/{project_name}/stats` 接口

修改统计查询,适配 Scanner V2 的图结构:

```python
@app.get("/api/projects/{project_name}/stats")
async def get_project_stats(project_name: str):
    """获取项目统计信息（支持 Repo 和 Project 节点）"""
    # 查询 Repo 节点 (Scanner V2)
    stats_query = """
    MATCH (r:Repo {name: $project_name})
    OPTIONAL MATCH (r)-[:CONTAINS]->(t:Type)
    OPTIONAL MATCH (t)-[:DECLARES]->(m:Method)  # ✅ 通过 Type 查询 Method
    OPTIONAL MATCH (t)-[:DECLARES]->(f:Field)   # ✅ 通过 Type 查询 Field
    OPTIONAL MATCH (t1:Type)-[c:CALLS]->(t2:Type)
    WHERE (r)-[:CONTAINS]->(t1) AND (r)-[:CONTAINS]->(t2)
    RETURN 
        count(DISTINCT t) as types,
        count(DISTINCT m) as methods,
        count(DISTINCT f) as fields,
        count(DISTINCT c) as total_calls
    """
```

## 兼容性说明

### 向后兼容

修改后的 API 同时支持:
1. **Scanner V2** 创建的 `Repo` 节点 (优先)
2. **Scanner V1** 创建的 `Project` 节点 (降级)

这确保了:
- ✅ 新扫描的项目 (使用 Scanner V2) 能正常显示
- ✅ 旧扫描的项目 (使用 Scanner V1) 仍然能正常显示
- ✅ 不需要重新扫描所有项目

### 迁移建议

1. **短期**: 使用当前的兼容方案,同时支持两种节点类型
2. **长期**: 
   - 逐步迁移所有项目到 Scanner V2
   - 删除旧的 `Project` 节点
   - 简化 API,只查询 `Repo` 节点
